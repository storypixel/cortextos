# Brief Format

Each selected signal gets one brief file at `research/output/YYYY-MM-DD/briefs/SLUG.md`.

## Fields

| Field | Description |
|-------|-------------|
| Title | Signal title from source |
| Date | Run date |
| Source | Source name and URL |
| Author | Author, channel, or repo |
| Category | User-configured category |
| Funnel stage | top / middle / bottom |
| Output type | short_form_video / long_form_post / competitive_brief / internal_note |
| Score | Weighted average, 1-10 |
| Executive summary | 2-3 sentences: what, why now |
| Evidence | Specific facts, numbers, quotes from source |
| Technical details | What is technically happening |
| Why it matters | 1-3 reasons tied to target audience |
| Content angles | 2-5 specific hooks or frames |
| Suggested hooks | Opening lines for content |
| Recommended action | create content / investigate / share internally / monitor / no action |
| Confidence | high / medium / low |
| Freshness | Published date, days old, shelf life |

## Slug Format

`SCORE-CATEGORY-TITLE-SLUG.md` -- e.g. `8.2-competitive-intel-composio-security-breach.md`

Lowercase, hyphens, no special characters, max 80 chars.
